# 数据库结构变更记录

> 相对于"数据库结构说明.md"（v1.0, 2026-06-27）的变更。

---

## 变更 1：user 表新增列

### vip_until — VIP 到期时间

| 属性 | 值 |
|------|-----|
| 字段名 | `vip_until` |
| 类型 | DATETIME |
| 默认值 | NULL |
| 说明 | 记录用户 VIP 到期时间。NULL 表示非 VIP 或已过期。替代了原来存储在 `profile` 和 `last_checkin` 中的 VIP 状态。 |

**迁移 SQL**：
```sql
ALTER TABLE user ADD COLUMN vip_until DATETIME NULL AFTER last_login;
```

**过渡策略**：
- 写操作：双写 `vip_until` 新列 + 旧 `profile`/`last_checkin` 列
- 读操作：优先读取 `vip_until`，若为 NULL 回退到 `profile` JSON 解析

### cefr_progress — CEFR 等级进度

| 属性 | 值 |
|------|-----|
| 字段名 | `cefr_progress` |
| 类型 | INT |
| 默认值 | 0 |
| 说明 | 记录用户 CEFR 等级进度（0-100）。替代了原来存储在 `profile` JSON 中的进度值。 |

**迁移 SQL**：
```sql
ALTER TABLE user ADD COLUMN cefr_progress INT DEFAULT 0 AFTER vip_until;
```

**过渡策略**：
- 写操作：双写 `cefr_progress` 新列 + 旧 `profile` JSON
- 读操作：优先读取 `cefr_progress`，若为 0 回退到 `profile` JSON 解析

|------|-----|

| 类型 | BIGINT |
| 约束 | PRIMARY KEY, AUTO_INCREMENT |

**迁移 SQL**：
```sql
ALTER TABLE admin_action_log ADD COLUMN id BIGINT AUTO_INCREMENT PRIMARY KEY FIRST;
```

## 变更 3：last_checkin 字段语义修正

**原语义**：同时存储"上次签到时间"和"VIP 到期时间"。

**新语义**：仅存储"上次签到时间"。VIP 到期时间迁移至 `vip_until` 列。

---

## 变更 4：config.properties 新增配置项

| 键 | 默认值 | 说明 |
|----|--------|------|
| `db.pool.maximumPoolSize` | 20 | HikariCP 最大连接数 |
| `db.pool.minimumIdle` | 5 | HikariCP 最小空闲连接 |
| `db.pool.idleTimeout` | 300000 | 空闲超时（ms） |
| `db.pool.maxLifetime` | 1200000 | 连接最大生命周期（ms） |
| `default.password` | 123456 | 管理员重置密码默认值 |
| `vip.cost.per.day` | 180 | VIP 每日消耗经验值 |
| `session.timeout.minutes` | 30 | 会话超时时间（分钟） |

JDBC URL 新增 `connectTimeout=10000&socketTimeout=10000`。

## 变更 5：DBUtil 改用 HikariCP 连接池

`DBUtil.java` 从 `DriverManager.getConnection()` 改为 HikariCP。接口不变，所有 DAO 无需修改。

## 变更 6：quiz_vocabulary / quiz_comprehension 表废弃

**日期**：2026-07

**变更内容**：`quiz_vocabulary`（词汇题库）和 `quiz_comprehension`（阅读理解题库）两张表已废弃。

**原因**：阅读理解题目改为 AI 实时生成，通过 `/api/article/quiz` 接口直接返回 JSON，不再持久化到数据库。原有的题目持久化、动态补充、软删除、投票评价等机制均已停用。

**影响**：
- `quiz_vocabulary` 和 `quiz_comprehension` 表作为冗余保留，业务代码不再读写
- `article` 表的 `vocquiz_num` 和 `comquiz_num` 字段不再维护
- 对应的投票表 `ai_vocquiz_vote` 和 `ai_comquiz_vote` 随之一并废弃
- 实体类 `QuizVocabulary.java` 和 `QuizComprehension.java` 仅保留用于可能的向下兼容

**清理建议**：可在后续大版本中通过数据库迁移脚本删除这四张废弃表。
