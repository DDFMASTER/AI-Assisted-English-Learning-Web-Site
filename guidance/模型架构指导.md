# AAEL 模型架构指导

## 1. 架构概述

AAEL 采用 **JSP Model2** 模式的变体，将应用程序划分为清晰的三层架构，实现了**关注点分离**（Separation of Concerns）。

## 2. JSP Model2 对照

AAEL 架构是 JSP Model2 的现代变体：

| JSP Model2 角色 | 原始实现 | AAEL 实现 | 说明 |
|-----------------|---------|-----------|------|
| **View** (视图层) | JSP 页面 | Vue (Vite 开发服务器) | 前端 SPA，通过 AJAX 与后端通信 |
| **Controller** (控制层) | Servlet | Servlet (Jakarta Servlet API) | 接收 HTTP 请求，调用 Service，返回 JSON |
| **Model** (模型层) | JavaBean + DAO | Service + Entities + DAO | 三层协作完成业务处理与数据持久化 |

### Model 层内部结构

在传统 JSP Model2 中，Model 层较为笼统。AAEL 将 Model 层进一步细化为三个子层：

| 子层 | 对应包 | 职责 |
|------|--------|------|
| **业务逻辑** | `Service` | 处理业务规则，编排 DAO 调用，校验输入 |
| **数据实体** | `Entities` | 数据库记录的 Java 对象表示 (POJO) |
| **数据访问** | `DAO` | 封装 SQL，执行 CRUD，返回 Entity 对象 |

## 3. 请求处理完整流程

以下以**用户查词**为例，展示一次完整的请求生命周期：

```
 浏览器 (Vue)                          后端 (Tomcat)
 ──────────                            ─────────────

 1. 用户在搜索框输入
    "abandon"

 2. Vue 发起 AJAX 请求
    GET /api/word/search
    ?word=abandon
    &studyPurpose=四级
         │
         ▼
 3.                          WordSearchServlet.doGet()
                              │
                              │ 获取参数 word, studyPurpose
                              │ 调用 wordService.searchWord()
                              │
                              ▼
 4.                          WordService.searchWord()
                              │
                              │ 校验参数非空
                              │ 调用 wordDAO.findByWordAndStage()
                              │
                              ▼
 5.                          WordDAOImpl.findByWordAndStage()
                              │
                              │ STAGE_TABLE.get("四级") → "cet_4"
                              │ SQL: SELECT id, word, ...
                              │      FROM cet_4 WHERE word = ?
                              │
                              ▼
 6.                          MySQL 返回结果集
                              │
                              │ 映射 ResultSet → List<WordBase>
                              │
                              ◄──────── 返回到 Service ────────
                              │
                              │ 判断结果是否为空
                              │ 若为空则查询 AI 词典作为兜底
                              │ 组装 WordSearchResult
                              │
                              ◄──────── 返回到 Servlet ────────
                              │
 7.                          Servlet 调用 result.toJson()
                              │
                              │ 设置 Content-Type: application/json
                              │ 写入 JSON 响应
                              │
         ◄────────────────────┘
         │
 8. Vue 接收 JSON 响应
    解析并渲染页面
```

## 4. 各层约束规则

### 4.1 Servlet 层（控制层）

```java
// ✅ 正确示例 — Servlet 只处理 HTTP 请求/响应
@WebServlet("/api/word/search")
public class WordSearchServlet extends HttpServlet {
    private final WordService wordService = new WordService();

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp)
            throws IOException {
        resp.setContentType("application/json;charset=UTF-8");

        String word = req.getParameter("word");
        String stage = req.getParameter("studyPurpose");

        WordService.WordSearchResult result = wordService.searchWord(word, stage);
        resp.getWriter().write(result.toJson());
    }
}
```

**Servlet 可以做的事：**
- 获取 HTTP 请求参数
- 调用 Service 方法
- 设置响应头（Content-Type、编码等）
- 将 Service 返回的结果写入响应流

**Servlet 不能做的事：**
- ❌ 直接使用 `Connection`、`PreparedStatement`、`ResultSet`
- ❌ 直接拼接或执行 SQL 语句
- ❌ 包含业务逻辑判断
- ❌ 直接调用 DAO 层

### 4.2 Service 层（业务逻辑层）

```java
// ✅ 正确示例 — Service 只做业务逻辑
public class WordService {
    private final WordDAO wordDAO = new WordDAOImpl();

    public WordSearchResult searchWord(String word, String studyPurpose) {
        // 1. 校验参数
        if (isBlank(word) || isBlank(studyPurpose)) { ... }

        // 2. 调用 DAO
        List<WordBase> results = wordDAO.findByWordAndStage(word, studyPurpose);

        // 3. 业务判断：命中则返回，未命中则查 AI 词典
        if (!results.isEmpty()) { ... }
        AiWordDic aiResult = wordDAO.findLatestAiDic(word);
        ...
    }
}
```

**Service 可以做的事：**
- 校验输入参数
- 调用多个 DAO 方法，编排调用顺序
- 执行业务规则判断
- 组装返回结果对象

**Service 不能做的事：**
- ❌ 直接使用 `Connection`、`PreparedStatement`、`ResultSet`
- ❌ 直接拼接或执行 SQL 语句
- ❌ 读取 HTTP 请求参数
- ❌ 写入 HTTP 响应

### 4.3 DAO 层（数据访问层）

```java
// ✅ 正确示例 — DAO 只做 SQL 执行和结果映射
public class WordDAOImpl implements WordDAO {
    private static final Map<String, String> STAGE_TABLE = Map.of(
            "初中", "junior",
            "高中", "senior",
            "四级", "cet_4",
            "六级", "cet_6",
            "考研", "postgraduate",
            "托福", "toefl"
    );

    @Override
    public List<WordBase> findByWordAndStage(String word, String stage) {
        String tableName = STAGE_TABLE.get(stage);
        String sql = "SELECT id, word, phonetic, translation FROM "
                   + tableName + " WHERE word = ?";

        try (Connection conn = DBUtil.getConnection();
             PreparedStatement ps = conn.prepareStatement(sql)) {
            ps.setString(1, word);
            try (ResultSet rs = ps.executeQuery()) {
                // 映射 ResultSet → List<WordBase>
            }
        }
    }
}
```

**DAO 可以做的事：**
- 执行 SQL 语句（通过 `PreparedStatement`）
- 将 `ResultSet` 映射为 Entity 对象
- 使用 `DBUtil.getConnection()` 获取数据库连接
- 使用白名单控制动态表名/列名（防 SQL 注入）

**DAO 不能做的事：**
- ❌ 包含业务逻辑判断
- ❌ 读取 HTTP 请求参数
- ❌ 写入 HTTP 响应
- ❌ 直接拼接用户输入到 SQL（必须用 `PreparedStatement` 参数化）

## 7. 设计原则总结

| 原则 | 说明 |
|------|------|
| **单一职责** | 每层只做一件事：Servlet=请求处理，Service=业务逻辑，DAO=数据访问 |
| **依赖方向** | Servlet → Service → DAO → Database，上层依赖下层，不可逆 |
| **不让 SQL 泄露到 Servlet** | 所有 SQL 语句必须封装在 DAO 实现类中 |
| **不让 HTTP 细节污染 Service** | Service 方法签名只接受业务参数，不接受 HttpServletRequest/Response |
| **白名单防注入** | 动态表名/列名使用 Map 白名单映射，禁止直接拼接用户输入 |
| **参数化查询** | 用户输入值一律通过 `PreparedStatement.setXxx()` 传参 |
