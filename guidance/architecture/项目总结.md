# EngliAI — 在线英语学习平台项目总结

---

## 一、项目背景

在全球化和信息化深度融合的时代背景下，英语作为国际通用语言，已经深入到教育、科研、商务、文化交流等各个领域。对于中国学生而言，英语阅读能力和词汇积累是语言学习的核心环节，然而传统英语教学模式面临诸多挑战：教材内容更新缓慢、学习资源获取不便、缺乏个性化学习路径、课后练习形式单一、学生难以获得即时反馈等。

本项目 **EngliAI（AI-Assisted English Learning）** 正是为解决上述问题而设计开发的一款在线英语学习平台。项目面向**教育行业**和**外语学习领域**，以网站形式提供快速便捷的访问和学习途径，用户无需安装任何客户端软件，通过浏览器即可随时随地开展英语学习。

平台深度融合了人工智能技术，借助大语言模型（DeepSeek API）为用户提供优质阅读素材和具有挑战性的练习题目。核心价值体现在以下几个方面：

1. **个性化阅读推荐**：根据用户的英语水平等级（CEFR）和学习阶段，智能匹配合适难度的英语文章，确保阅读材料既具有挑战性又不至于过于困难，使用户始终处于最佳学习区。

2. **AI 辅助深度阅读**：阅读过程中提供单词释义查询、段落翻译、文化背景讲解、AI 例句生成等功能，帮助用户深入理解文章内容，扩展文化视野，而非停留在浅层的单词记忆层面。

3. **科学的词汇量评估**：采用基于 COCA 词频表的分层抽样算法和伪词诱饵机制，能够较为准确地评估用户的词汇量水平，为个性化学习提供数据支撑。

4. **自适应测评体系**：基于 CEFR（欧洲语言共同参考框架）标准，通过 AI 动态生成测评题目并评估用户的多维度能力（词汇、语法、阅读、文化、逻辑），帮助用户清晰认知自身英语水平并及时调整学习策略。

5. **游戏化激励机制**：通过每日任务、经验值（XP）获取、VIP 兑换等机制，结合英语天地中的情景对话、真人PK、单词接龙等趣味游戏，激发用户的学习动力，将枯燥的语言学习转变为富有成就感的持续过程。

本项目可有效辅助学生提高英语阅读能力和词汇量，适用于中学英语课外阅读、大学四六级备考、考研英语训练、托福备考等多种学习场景。同时，平台的管理后台为教师或管理员提供了内容管理和用户监控的便捷手段，使平台能够灵活适应不同教学管理需求。

## 二、项目内容

EngliAI 平台围绕"通过阅读提升词汇量和理解能力"这一核心目标，构建了从用户水平评估、个性化内容推荐、沉浸式阅读学习、游戏化练习巩固、自适应测评到学习趋势追踪的完整学习闭环。各功能模块之间层次分明、紧密关联。

### 2.1 用户系统与水平评估

#### 2.1.1 注册与登录

首次访问平台的用户为**游客**身份，可浏览文章列表但不能使用交互功能。用户需注册账户并登录后才能完整体验平台功能。注册时需填写用户名（支持中英文，最长 16 字符）、密码（采用加盐 MD5 加密存储），并选择当前学习阶段（初中 / 高中 / 四级 / 六级 / 考研 / 托福）。系统还提供了默认密码账户创建功能，方便管理员批量导入用户。

> **数据模型演进**：早期 VIP 状态存储在 `profile` JSON 字段中。现已迁移为独立数据库列 `vip_until`（DATETIME 类型，记录 VIP 到期时间）和 `cefr_progress`（INT 类型，记录 CEFR 晋级进度百分比），查询更高效、索引友好。旧 `profile` JSON 字段仍保留作为兼容过渡。

#### 2.1.2 首次词汇量快速测试

用户首次注册后，系统自动弹出**词汇量快速测试**弹窗（FirstVocabTestModal）。该测试以交互式卡片形式展示一系列英语单词，用户通过点击"认识"或"不认识"按钮作答。测试采用基于 COCA 20000 词频表的分层抽样算法，从 9 个词频段各抽取 7-13 个真实单词，同时混入约 7-13 个形似真词的伪词诱饵。通过伪词命中率矫正过度自信偏差，最终估算用户的词汇量，并将结果（词汇量数值及对应 CEFR 等级）与后续文章推荐和测评系统绑定。

#### 2.1.3 用户角色体系

系统设有三种用户角色：

| 角色 | 说明 |
|------|------|
| **普通用户（normal）** | 可阅读文章、查词、做测验、使用基本 AI 功能（有每日次数限制） |
| **VIP 用户（vip）** | 除普通用户功能外，AI 功能无次数限制，可主动调用 AI 生成新内容 |
| **管理员（admin）** | 拥有独立管理后台，可管理用户、文章，查看系统日志 |

普通用户可通过完成每日任务获取经验值（XP），累计足够的 XP 后可在个人中心兑换 VIP 身份（180 XP/天）。

### 2.2 智能读物匹配与阅读器

#### 2.2.1 读物匹配页

登录后进入**读物匹配页**（MaterialsPage），系统根据用户的 CEFR 水平等级自动筛选对应难度的文章列表。每篇文章以卡片形式展示，包含标题、来源、难度标签、点赞数等信息。用户可以按难度等级手动筛选文章。页面顶部设有搜索框，支持对文章标题进行防抖自动搜索，并可调用 AI 对搜索结果进行智能释义推荐。

#### 2.2.2 AI 沉浸式阅读器

点击文章进入**阅读器页面**（ReaderPage），这是平台的核心学习界面。阅读器提供以下交互功能：

- **单词查询**：点击文章中的任意单词，系统根据用户当前学习阶段查询对应词库表（初中/高中/四级/六级/考研/托福共六张静态词库），以弹窗展示单词的音标和释义。若当前阶段词库未命中，提示用户该词超出当前阶段，VIP 用户可进一步使用 AI 详解功能。

- **AI 单词详解**：当词库未命中或 VIP 用户主动请求时，后端先查询 AI 动态词典（ai_word_dic 表），未命中则调用 DeepSeek API 生成精讲内容（含音标、释义、详细解释），存入词典并返回。

- **AI 例句生成**：点击单词弹窗中的"生成例句"按钮，AI 为该单词生成 3 个包含中文翻译的例句，帮助用户在上下文中理解单词用法。

- **段落翻译**：用户划选文章中的句子或段落（超过 3 个单词），系统自动判断为段落类型，调用 AI 进行中英文翻译。翻译结果缓存在浏览器 IndexedDB 中，同设备重复访问时直接读取。

- **文化背景讲解**：阅读器侧边栏提供"文化背景分析"功能，AI 分析文章中的文化知识点（如节日、习俗、历史事件等），以结构化卡片形式展示，每张卡片包含标题和中英文内容。

- **阅读理解出题**：AI 基于文章内容生成阅读理解选择题，包含题目、选项、正确答案和解析。题目持久化到数据库，后续用户可复用。非 VIP 用户每日限制 3 篇文章的 AI 出题服务。

- **文章整体讲解**：AI 对文章进行整体解读，概括篇章大意，分析文章结构和写作特点。用户可对讲解内容点赞/踩/取消。

### 2.3 阅读完成与学习记录

#### 2.3.1 阅读完成机制

平台采用**自然的阅读完成判定**：用户将文章滚动至底部（阅读进度达到 100%）即视为完成阅读。系统通过监听页面滚动事件实时计算阅读进度，当用户划到底部时自动记录学习完成状态。这种方式更贴近真实的阅读习惯，避免了强制测验带来的学习中断感。

#### 2.3.2 阅读理解出题（可选辅助练习）

测验不再是完成阅读的强制条件，而是作为**可选的辅助练习工具**。用户可以在阅读器侧边栏自主调出阅读选择题进行练习。题目由 AI 基于文章内容实时生成（通过 `/api/article/quiz` 接口），以 JSON 格式直接返回前端，**不再持久化到数据库**。每题包含 4 个选项、正确答案和解析，用户答题后可查看解析。非 VIP 用户每日限制 3 篇文章的 AI 出题服务。

> **历史备注：** 早期版本的 `quiz_vocabulary`（词汇题库）和 `quiz_comprehension`（阅读理解题库）表已废弃，作为冗余表保留在数据库中。原有的题目持久化、动态补充、软删除、投票评价等机制均已停用，改为"实时生成、用完即弃"模式。

#### 2.3.4 学习记录

用户完成文章阅读后，系统记录学习完成状态，构成用户学习历史的基础数据，同时避免重复推荐已学文章。

### 2.4 自适应测评中心

**测评中心**（AssessmentPage）是用户检验学习成果、获取进阶反馈的核心模块：

1. **首题即时返回**：用户开始测评后，AI 生成首道题目并立即返回（约 15-20 秒），让用户快速进入答题状态。

2. **后台异步出题**：剩余 9 道题目由 AI 在后台异步生成，前端通过轮询机制每 3 秒拉取新题目，生成一道展示一道。

3. **难度自适应**：出题难度基于用户当前 CEFR 等级（从词汇量测试结果推算），确保题目难度与用户水平匹配。若用户水平提升，下一轮测评的题目难度也会相应调整。

4. **答案去偏**：为防止连续 3 题答案相同影响测评效度，系统会在 AI 出题时注入约束条件，避免答案分布不均。

5. **AI 多维评估**：提交全部答案后，AI 从词汇（vocabulary）、语法（grammar）、阅读（reading）、文化（culture）、逻辑（logic）五个维度对用户表现进行评估，给出综合得分和 CEFR 等级。

6. **CEFR 晋级机制**：单次测评最高增加 20% 进度（满分 100 分即 `/5 = 20%`），分数低于 30 分不增加进度。进度达到 100% 时自动晋升一个 CEFR 等级（如 B1 → B2）。测评结果以**雷达图**和详细报告形式在结果页（ResultPage）展示。

### 2.5 英语天地（游戏化学习）

**英语天地**（EnglishWorldPage）是平台的游戏化学习模块，提供三种趣味英语游戏，让用户在游戏中提升英语能力。该模块通过丰富的交互设计和 AI 辅助，将语言学习融入沉浸式游戏体验。

#### 2.5.1 情景对话

用户选择场景后，AI 扮演虚拟角色与之进行英语对话，模拟真实交流场景：

- **四大场景**：购物（🛍️ Shop Assistant）、旅游（✈️ Hotel Receptionist）、点餐（🍽️ Waiter）、职场面试（💼 HR Manager）
- **双语旁白**：AI 以中英文双语提供场景推进旁白，引导对话走向
- **角色扮演**：AI 以角色身份进行英语对话，用户以英语回复
- **回复建议**：界面右侧实时提供回复建议，帮助用户找到合适的英语表达
- **智能纠偏**：当用户偏离场景或出现严重语法/拼写错误时，AI 发出场景偏离警告（🚫）或英语提醒（⚠️）
- **底层支持**：后端通过 `/api/roleplay/chat` 接口调用 DeepSeek API，前端在 API 不可用时自动回退到本地模拟对话

#### 2.5.2 真人 PK

用户可与好友实时对战，比拼词汇量和答题速度：

- **房间系统**：创建公开或私密房间，通过 4 位数字房间码邀请好友加入
- **难度选择**：支持初中/高中/四级/六级/考研/托福六个难度等级
- **对战机制**：每局 20 道词汇选择题，每题限时 15 秒，答对得分，超时或答错不得分
- **实时进度**：双人进度条实时显示双方答题进度（已答数/正确数）
- **胜负判定**：正确数多者获胜；正确数相同时耗时短者获胜；中途退出者判负
- **轮询机制**：前端通过 2 秒间隔轮询对手状态，房主操作（调整难度、开始游戏、解散房间）实时同步
- **底层支持**：后端通过 `/api/pk/room`（房间管理）和 `/api/pk/game`（对局管理）接口，使用内存级 GameRoomManager 管理房间生命周期，定期清理过期房间

#### 2.5.3 单词接龙

AI 与用户轮流进行英语单词接龙，首尾字母相接：

- **标准模式**：总时长 5 分钟，AI 响应期间计时暂停，适合休闲练习
- **无尽模式**：无总时长限制，但每轮仅有 8 秒思考时间，超时即结束，适合挑战极限
- **规则约束**：单词至少 3 个字母、不可重复使用、首字母须与上一单词尾字母一致
- **进度展示**：实时显示接龙链条（颜色区分 AI/用户）、用户贡献单词数、倒计时进度条
- **底层支持**：后端通过 `/api/wordchain` 接口进行单词合法性验证和 AI 单词生成，前端在 API 不可用时自动回退到本地词库

### 2.6 个人中心

**个人中心**（ProfilePage）汇聚用户的学习数据和个性化设置：

- **词汇量详细测试**：用户可随时在个人中心进行更详细的词汇量测试。该测试同样采用分层抽样算法，AI 生成释义选项和伪词诱饵，提供比首次快速测试更准确的词汇量评估结果。支持"更多单词"功能，每次额外生成 10 个测试词，测试越多结果越精确。

- **学习趋势**：以图表形式展示用户的每日在线学习时长记录，支持设置目标学习时间。系统通过会话监听器和 IndexedDB 自动追踪用户的在线时长。

- **学习记录**：展示用户已完成的文章列表和完成时间，方便回顾学习历程。

- **生词本**：用户在阅读过程中查询的单词自动加入生词本（存储于 IndexedDB），最大容量 200 词，支持随时查看和复习。

- **单词复习**（VocabReviewModal）：提供交互式单词复习模态框，支持按阶段筛选复习范围，以卡片翻页形式展示单词和释义，帮助用户巩固词汇记忆。

- **错题集与导出**：用户在测验和测评中答错的题目自动记录，支持查看错题和正确答案。提供**错题导出**功能，可将错题以文本格式导出保存，方便离线复习。

- **设置**：可修改目标学习时间、当前学习阶段（不影响英语水平，但影响词书查询逻辑）。支持自定义头像，内置多种头像选项。

- **经验值与 VIP 兑换**：展示当前 XP 余额，支持使用 XP 兑换 VIP 天数（180 XP/天）。

- **文章点赞**：在阅读器页面（ReaderPage）中，文章标题旁设有心形点赞按钮，调用 `POST /api/article/like` 接口记录用户对文章的喜爱，方便其他用户发现优质内容。

- **阅读时间估计**：文章卡片上展示"X min read"阅读时间预估，由 `ReadingTimeUtil` 基于文章字数计算，精度为 0.5 分钟，帮助用户合理安排阅读计划。

- **快速测试差异化**：首次词汇量测试弹窗（FirstVocabTestModal）通过 `isInitial` prop 区分首次测试和重测场景——首次使用显示引导性欢迎文案，重测时提示"再次测试可获得更精确结果"。

### 2.7 每日任务与签到

- **每日签到**：用户每天可签到一次，获得 10 点经验值。
- **每日任务**：系统设置每日学习任务（如完成一篇文章阅读、完成一次测评、进行一次单词复习），完成后可获取额外的经验值奖励。

### 2.8 管理后台

**管理后台**（AdminPage）仅供 admin 角色用户访问，提供以下功能：

- **用户管理**：查询所有用户列表（含用户名、角色、学习阶段、词汇量、经验值、注册时间等），支持删除普通用户（级联清理投票记录和学习记录），支持重置用户密码为默认密码。

- **在线用户监控**：查看当前在线用户列表，支持对普通用户执行**强制下线**操作（通过 kickedUserIds 集合实现）。

- **文章管理**：支持增删查改文章（标题、正文、来源、难度等级），删除文章时级联清理关联的学习记录和投票数据。

- **IP 请求日志**：通过请求日志监听器（RequestLogListener）记录所有 API 请求的 IP 地址、请求路径、耗时等信息，管理员可在后台查询和分析。支持日志的**分页查询**、**按 IP 筛选**、**全部清空**、**选中删除**以及**数据备份**功能，便于安全审计和异常流量排查。

### 2.9 功能关联总结

各功能模块之间形成了紧密的学习闭环：

```
注册/登录 → 词汇量快速测试 → 读物匹配 → 沉浸式阅读（滚到底即完成）
    │                         │              │
    │                         │              └──→ 可选测验练习
    │                         │                      
    │                         ├──→ 英语天地 ←──┐
    │                         │    ├─ 情景对话   │
    │                         │    ├─ 真人PK    │  游戏化
    │                         │    └─ 单词接龙   │  激励
    │                         │                │
    ↓                         ↓                │
个人中心 ←── 测评反馈 ←── 自适应测评             │
    │                                           │
    └── 经验值/VIP ←── 每日签到与任务 ←─────────┘
```

#### 3.2.1 Web 层（Servlet + Filter + Listener）

Web 层是 MVC 架构中的 **Controller**，负责接收前端 HTTP 请求、进行权限校验、调用业务逻辑层处理数据、返回 JSON 响应。

- **Servlet 控制器**：平台共包含 40+ 个 Servlet 类，每个 Servlet 负责一组相关的 API 端点。如 `UserLoginServlet` 处理登录请求、`ArticleDetailServlet` 返回文章详情、`AssessmentGenerateServlet` 触发 AI 出题、`RolePlayServlet` 处理情景对话、`PkRoomServlet` 和 `PkGameServlet` 管理真人PK、`WordChainServlet` 处理单词接龙等。

- **LoginFilter（登录过滤器）**：拦截所有非公开 API 请求，验证用户身份。从请求头或 Session 中提取用户 token，校验通过后将请求转发给目标 Servlet，未通过则返回 401。

- **SessionListener（会话监听器）**：监听 HTTP 会话的创建和销毁事件，维护在线用户计数（通过 AtomicInteger）和用户-会话 ID 映射（通过 ConcurrentHashMap），为管理员后台提供在线用户列表数据。

- **RequestLogListener（请求日志监听器）**：记录每个 HTTP 请求的 IP 地址、请求方法、请求路径、响应状态码和耗时，存储到应用上下文的线程安全列表中（上限 **10000** 条，FIFO 淘汰），供管理员查询分析。支持分页查询、IP 筛选、清空、删除和备份操作。**日志持久化**：支持 JSON 文件持久化存储，每日自动轮转归档（昨日日志重命名为 `request-logs-YYYY-MM-DD.json`），通过 `MonitorService` 实现周期性自动保存（默认每 5 分钟）、Tomcat 启动时自动加载历史日志、过期归档文件自动清理（默认保留 7 天）。

- **AppContextListener（应用上下文监听器）**：在应用启动时初始化全局属性（在线计数器、用户会话映射、被踢用户集合、启动时间戳、PK 房间 Map），在应用关闭时清理资源。

#### 3.2.2 业务逻辑层（Service）

业务逻辑层是 MVC 架构中的核心处理层，封装了所有业务规则和流程控制：

- **UserService**：处理用户注册、登录、签到、经验值累加、VIP 兑换、VIP 过期检测等核心用户业务流程。密码使用加盐 MD5 哈希存储，VIP 到期后自动降级。

- **AIService**：封装所有 AI 相关功能的调用逻辑，统一通过 DeepSeek API 进行通信。核心职责包括：

  - **单词例句生成**：调用 DeepSeek，传入 system prompt（`word-examples.txt`）和单词，要求返回结构化 JSON（含 en/zh 字段对）。
  - **段落翻译**：传入翻译专用 prompt 和段落文本，解析返回的 translation 字段。
  - **文化背景分析**：传入文章全文和 culture-analysis prompt，解析 title/content/zh 三元组。
  - **阅读理解出题**：传入文章和 quiz-generation prompt，解析 question/optionA-D/answer/explanation 字段。
  - **测评出题**：分单题快速版本和批量生成版本，根据学习阶段（studyPurpose）生成适配难度的短文章和选择题。
  - **测评评估**：传入题目及正确答案 + 用户答案，AI 返回多维度评分和 CEFR 等级。
  - **AI 标题生成**：为无标题文章自动生成英文标题。
  - **AI 查词**：当所有词书未命中时，由 AI 搜索单词释义，结果存入 AI 动态词典。
  - **情景对话**：处理角色扮演对话，生成 AI 角色回复、双语旁白和回复建议。
  - **单词接龙验证**：验证用户输入的单词合法性（拼写、首字母匹配、是否重复）。

  所有 AI 调用的流程统一：
  1. 构建 system prompt + user content
  2. 通过 Java 内置 `java.net.http.HttpClient` 发送 POST 请求到 DeepSeek API
  3. 接收 JSON 响应，提取 choices[0].message.content
  4. 规范化 JSON（去除 markdown 代码块包裹、压缩空白字符）、解析字段
  5. 封装为结构化结果对象返回

- **VocabTestService**：词汇量测试的核心算法服务。加载 COCA 20000 词频数据，将单词按频率分为 9 个频段（101-500、501-1500、...、16001-20000），同时加载 500 个伪词。采用分层随机抽样算法，从每个频段抽取 7-13 个真实词，混合 7-13 个伪词，共约 100 个测试词。估算算法：对各频段计算矫正认识率 r_i = max(0, p_i_raw − 0.5 × pf)，再将矫正率外推到频段总词数求和，加 100 个默认功能词，得出总词汇量估计值；同时使用二项分布方差汇总计算 95% 置信区间。

- **GameRoomManager**：真人 PK 房间管理服务。以 ConcurrentHashMap 管理所有活跃房间，支持创建房间（生成 4 位随机房间码）、加入房间、解散房间、更新难度、查询房间列表（分页）等操作。通过定时任务清理超过 30 分钟无活动的过期房间。

- **AdminService**：管理员业务逻辑，包含创建/删除用户、重置密码、增删改文章等操作。删除用户和文章时使用数据库事务保证级联清理的原子性。所有管理员关键操作记录到 `admin_action_log` 审计日志表。

- **LoginAttemptService**：登录限流服务，基于 IP 和用户名记录失败尝试次数，超过阈值后临时锁定，防止暴力破解。

- **ReadingTimeUtil**：阅读时间估算工具，根据文章字数计算预估阅读时长（精度 0.5 分钟），在前端文章卡片上展示"X min read"。

- **VocabTestCardService**：词汇量测试卡片服务，负责基于 COCA 词频表生成交互式测试卡片，支持分层抽样和伪词混入。

- **ArticleService / WordService / MonitorService / HealthService**：分别负责文章推荐与匹配、词库查询与维护、系统监控、健康检查等业务。

### 3.5 算法说明

#### 3.5.1 词汇量分层抽样估计算法

**原理**：基于 COCA（Corpus of Contemporary American English）20000 词频表，将英语单词按使用频率从高到低排序，分为 9 个频段。每个频段的总词数已知（如 101-500 名含 400 词）。测试时从每个频段随机抽取 7-13 个样本单词，混入约 7-13 个伪词（英语化但实际不存在的单词），用户逐个判断是否认识。

**估算公式**：

$$Vocabulary \approx 100 + \sum_{i=1}^{9} N_i \times \max\left(0, \frac{k_i}{n_i} - \frac{1}{2} \times \frac{k_f}{n_f}\right)$$

其中 $N_i$ 为第 i 频段的总词数，$k_i/n_i$ 为该频段的原始认识率，$k_f/n_f$ 为伪词命中率（代表用户过度自信程度）。95% 置信区间基于各频段二项分布方差汇总后取 ±1.96σ 计算。

**AI 辅助增强**：AI 生成词汇测试选项时，为每个真实单词生成 4 个中文释义选项（含 1 个正确释义和 3 个干扰项），为伪词生成相似但不存在的释义，进一步提高测试准确性。

#### 3.5.2 CEFR 等级映射算法

用户词汇量测试结果与 CEFR 等级的映射关系：

| CEFR 等级 | 对应词汇量 | 中文标签 |
|-----------|-----------|---------|
| A1 | 500 | 初级 |
| A2 | 1500 | 初级上 |
| B1 | 3000 | 中级 |
| B2 | 5000 | 中高级 |
| C1 | 8000 | 高级 |
| C2 | 12000 | 精通 |

#### 3.5.3 伪词矫正机制

在词汇量测试中，用户对伪词的误判率用于矫正对真实词的过度自信。矫正系数为 0.5，即从各频段原始认识率中扣除伪词命中率的一半，避免高估。

#### 3.5.4 渐进式题目生成与轮询

测评模块采用"首题即时返回 + 后台异步生成 + 前端轮询拉取"的策略：AI 首先快速生成首道题目（约 15-20 秒）让用户立即进入答题状态，剩余 9 道题在后台异步生成，前端每 3 秒轮询一次拉取新题。这种设计将用户感知的等待时间压缩到首题生成时间，大幅提升体验。

#### 3.5.5 真人PK房间管理与状态同步

PK 房间使用 ConcurrentHashMap 存储于应用上下文，通过 2 秒间隔的 HTTP 轮询实现双方状态同步（答题进度、正确数、耗时）。房间采用 4 位数字房间码，支持公开（列表可见）和私密（仅房间码加入）两种模式。定时任务每 5 分钟清理超过 30 分钟无活动的过期房间，防止内存泄漏。

## 四、项目日志

### 第一周（项目启动与规划）

首周完成了项目的整体规划与准备工作。团队进行了详细的需求讨论，明确了项目定位——打造一款融合 AI 技术的在线英语学习平台。在此基础上完成了人员分工，确立了前后端分离的开发模式。环境配置方面，搭建了 IntelliJ IDEA 开发环境，配置了 Maven 项目结构和 Tomcat 10 本地运行环境，并在阿里云 ECS 上完成了云服务器的初始配置。同时在 GitHub 上建立了代码仓库，制定了 Git Flow 分支管理策略，为后续协作开发奠定基础。

### 第二周（前端框架转型与基础设施）

本周完成了初步的 UI 页面设计，在服务器上搭建了 Nginx 反向代理 + Tomcat 的运行架构，并完成了 MySQL 8.4 数据库的表结构设计。经过技术调研，团队决定使用 Vue 3 框架取代传统的 JSP 前端开发方式，以提升用户交互体验和开发效率。这一决策带来了核心技术挑战：如何将 Vue 的单页应用模式与现有的 Tomcat Servlet 架构进行融合。最终采用了 Nginx 统一服务静态资源（Vue SPA）并反向代理 API 请求至 Tomcat 的方案，实现了前后端的有效解耦。另一个难题是服务器 JDK 21 与本地开发环境 JDK 版本不一致，通过在 Maven 中指定编译目标版本为 21 解决了兼容性问题。

### 第三周（后端功能完善与 AI 集成）

本周重点完善了后端业务功能，设立了 LoginFilter 登录过滤器、SessionListener 会话监听器和 RequestLogListener 请求日志监听器，构建了完整的请求处理链路。前端进一步丰富了页面交互细节，并正式接入了 DeepSeek API 实现 AI 释义、翻译等智能化功能。开发过程中遇到若干技术难点：Servlet 中方法调用次序错误导致数据库读写顺序与预期不符，引发 HTTP 500 错误，通过梳理请求生命周期和重构代码执行流程解决了该问题；AI 查词响应缓慢——每次查询需等待 API 完整返回——为此建立了 ai_word_dic 动态词典缓存机制，首次查询后持久化到数据库，后续命中直接返回，大幅降低了响应延迟；此外，用户会话 Session 过期后，测评进度无法正常更新的问题，通过前端增加会话有效性检测和自动重试机制得以解决。

### 第四周（游戏功能实现与安全加固）

本周正式完成了英语天地模块的三大游戏功能——情景对话、真人 PK 和单词接龙，通过 GameRoomManager 内存管理和轮询机制实现了实时对战体验。与此同时，对前端结构进行了全面重构，将原本集中于单一大型 Vue 文件中的业务逻辑拆分和解耦为多个独立组件和状态管理模块，显著提升了页面响应速度和代码可维护性。安全方面，针对发现的多项隐患进行了集中修复：过时的 Session 仍持续发出请求导致在线人数统计和 IP 请求日志数据失真，通过增强 SessionListener 的失效检测逻辑予以更正；公开的 `/api/` 路径存在 URL 劫持风险，在 Nginx 和 Filter 层面添加了请求合法性校验；原先硬编码在前端代码中的敏感 API 端点存在泄露风险，统一迁移至后端配置并通过环境变量管理。同时为管理后台扩充了 IP 日志的筛选、备份和一键清空等功能，增强了系统的可运维性。
