# 未来扩展空间

> 本文档梳理当前数据库结构中标记为"已废弃/待扩展"的表和字段，分析其潜在复用价值与扩展方向。
> 关联文档：`数据库结构说明.md`（v2.3）。

---

## 一、概述

项目发展过程中，部分数据库表和字段因架构调整而停止使用，但表结构完整、数据模型清晰，具备复用潜力。与其直接删除，不如将其视为**预置基础设施**，为未来功能迭代保留可能性。

当前涉及的废弃对象：

| 类型 | 对象 | 废弃原因 |
|------|------|---------|
| 表 | `quiz_vocabulary` | 题目改为 AI 实时生成，不再持久化 |
| 表 | `quiz_comprehension` | 同上 |
| 表 | `ai_vocquiz_vote` | 父表废弃，投票无对象 |
| 表 | `ai_comquiz_vote` | 同上 |
| 字段 | `article.vocquiz_num` | 题库表废弃，计数不再维护 |
| 字段 | `article.comquiz_num` | 同上 |

---

## 二、逐对象扩展分析

### 2.1 `quiz_vocabulary` — 词汇题库

**当前状态**：冗余保留，含 `word`、`article_id`、`content`（JSON：题干 + 4 选项 + 答案 + 解析）、投票计数、软删除标记。

**可扩展方向**：

| 方向 | 描述 | 改动量 |
|------|------|--------|
| **题目缓存复用** | 对高频访问的文章，AI 生成后写入此表。后续用户访问同一文章时直接读取，减少 AI 调用成本和延迟。需配合 Redis 或简单的 TTL 过期策略 | 中 |
| **社区出题** | 允许 VIP 用户或教师为文章手动编写词汇题，存入此表。`content` 字段的 JSON 格式已为此预留扩展性 | 中 |
| **错题集升级** | 当前错题集仅存储于前端 IndexedDB。可将用户答错的题目关联到此表，实现服务端错题同步与跨设备恢复 | 大 |
| **词汇题难度分级** | 在 `content` JSON 中新增 `difficulty` 字段，按 CEFR 或学习阶段分级，提供差异化练习 | 小 |
| **出题质量分析** | 重新激活 `like_count` / `dislike_count` 和 `ai_vocquiz_vote`，统计题目质量，淘汰低质题目 | 小 |

**建议优先级**：题目缓存复用 > 错题集升级 > 社区出题

### 2.2 `quiz_comprehension` — 阅读理解题库

**当前状态**：冗余保留，结构与 `quiz_vocabulary` 类似，不含 `word` 字段。

**可扩展方向**：

| 方向 | 描述 | 改动量 |
|------|------|--------|
| **阅读理解缓存** | 与词汇题缓存同理，对热门文章缓存 AI 生成的阅读理解题 | 中 |
| **分层题目库** | 为同一文章生成多套不同难度的题目（初级/中级/高级），用户可自选难度 | 中 |
| **教师题库** | 教师/管理员通过后台手动编写并绑定阅读理解题，替代或补充 AI 生成 | 小 |
| **测评中心对接** | 将缓存的阅读题整合到测评中心（AssessmentPage），作为自适应测评的备选题目源 | 大 |

**建议优先级**：教师题库 > 阅读理解缓存 > 测评中心对接

### 2.3 `ai_vocquiz_vote` / `ai_comquiz_vote` — 题目投票表

**当前状态**：冗余保留，联合主键 `(user_id, content_id)`，三态 action（0/1/2）。

**可扩展方向**：

| 方向 | 描述 | 改动量 |
|------|------|--------|
| **随父表重新激活** | 当 `quiz_vocabulary` / `quiz_comprehension` 被重新使用时，投票表自然恢复功能 | 小 |
| **统一投票表** | 将四张独立的投票表（`ai_word_vote`、`ai_article_explain_vote`、`ai_vocquiz_vote`、`ai_comquiz_vote`）合并为一张通用投票表，使用 `target_type` + `target_id` 区分对象（参考 `admin_action_log` 的设计模式） | 大 |
| **质量排序** | 基于投票数据对题目进行质量排序，优先展示高分题目 | 小 |

**建议优先级**：随父表激活 > 质量排序 > 统一投票表

### 2.4 `article.vocquiz_num` / `article.comquiz_num` — 题目计数

**当前状态**：不再维护，默认值 0。

**可扩展方向**：

| 方向 | 描述 | 改动量 |
|------|------|--------|
| **缓存命中计数** | 当题目缓存重新激活时，这两个字段可用作"已缓存题目数"的快速查询计数器 | 小 |
| **题目丰富度指标** | 作为文章"练习资源丰富度"的指标，帮助用户选择有更多练习资源的文章 | 小 |
| **直接移除** | 如果确定不再使用，可通过迁移脚本删除，简化 article 表结构 | 小 |

**建议**：暂不删除，待题目缓存方案确定后再决定去向。

## 三、其他潜在扩展空间

以下为基于现有基础设施的独立扩展思路，不限于废弃表。

### 3.1 用户体系

| 方向 | 涉及表/字段 | 描述 |
|------|-----------|------|
| 学习阶段细化 | `user.study_purpose` | 当前仅 ENUM 6 个值，可扩展为自定义标签或支持多阶段并行学习 |
| 个人主页丰富化 | `user.profile` | VARCHAR(127) 可改为 TEXT，支持富文本自我介绍 |
| 学习统计数据 | 新建 `user_stats` 表 | 累计阅读字数、总学习时长、连续签到天数等聚合统计 |
| 社交功能 | 新建 `user_follow` 表 | 用户关注/粉丝关系，学习动态分享 |

### 3.2 文章体系

| 方向 | 涉及表/字段 | 描述 |
|------|-----------|------|
| 文章标签系统 | 新建 `article_tag` + `tag` 表 | 多标签分类（科技/文化/体育…），替代当前单一的 difficulty 分类 |
| 文章评论 | 新建 `article_comment` 表 | 用户对文章的评论/讨论，可包含划线批注 |
| 文章难度细化 | `article.difficulty` | 当前为 ENUM，可扩展为独立的难度等级表（如 CEFR + 词数 + 句式复杂度） |
| 系列文章 | `article` + 新建 `article_series` | 多篇文章组成系列课程，按顺序推荐学习 |

### 3.3 AI 功能

| 方向 | 涉及表/字段 | 描述 |
|------|-----------|------|
| AI 出题批量调度 | — | 后台定时对无题文章批量调用 AI 出题并缓存到 quiz 表 |
| AI 翻译持久化 | 新建 `article_translation` 表 | 当前段落翻译仅缓存在浏览器 IndexedDB，可服务端持久化供所有用户复用 |
| 多模型支持 | `config.properties` | 支持切换不同 AI 模型（DeepSeek / GPT / Claude），记录每次调用的模型和耗时 |

### 3.4 游戏化

| 方向 | 涉及表/字段 | 描述 |
|------|-----------|------|
| PK 历史战绩 | 新建 `pk_history` 表 | 持久化对战记录（双方 ID、得分、耗时、胜负） |
| 排行榜 | 新建 `leaderboard` 表或 Redis | 周榜/月榜/总榜，按词汇量、经验值、PK 胜率排名 |
| 成就系统 | 新建 `achievement` + `user_achievement` | 解锁条件（如"连续签到 7 天"、"词汇量突破 5000"），奖励 XP |

### 3.5 数据与分析

| 方向 | 涉及表/字段 | 描述 |
|------|-----------|------|
| 学习分析仪表盘 | 聚合查询现有表 | 用户维度的学习热力图、薄弱词频段分析、推荐难度曲线 |
| 内容质量监控 | `admin_action_log` 扩展 | 自动标记低质内容（高踩率 AI 解释、零点赞文章），触发管理员审查 |

## 四、扩展优先级建议矩阵

```
                  高价值
                    │
    题目缓存复用     │    错题集服务端同步
    教师题库         │    学习分析仪表盘
    PK 历史战绩      │
                    │
  ──────────────────┼────────────────── 低改动成本
                    │
    文章标签系统     │    统一投票表
    社区出题         │    社交功能
    成就系统         │
                    │
                  低价值
```

**短期（1-2 周可实现）**：
1. 重新激活 `quiz_vocabulary` / `quiz_comprehension` 作为 AI 出题缓存
2. `article.vocquiz_num` / `comquiz_num` 作为缓存计数器
3. PK 历史战绩持久化

**中期（1-2 月）**：
4. 错题集服务端同步
5. 教师题库后台
6. 文章标签与评论

**长期（规划中）**：
7. 统一投票表重构
8. 社交功能与成就系统
9. 学习分析仪表盘
