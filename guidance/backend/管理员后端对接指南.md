# 管理员后端对接指南

---

## 一、概述

本文档供前端开发人员参考，说明管理员功能涉及的后端 API 端点、请求参数、响应格式，以及 Vue 页面设计建议。

管理员身份由 `user` 表的 `role` 字段控制，取值为 `ENUM('normal','vip','admin')`。用户登录成功后，响应中会携带 `role` 字段，前端据此判断是否展示管理入口。

---

## 二、管理员身份识别

### 登录接口回顾

**`POST /api/user/login`**

请求（form-urlencoded）：

| 参数 | 类型 | 说明 |
|---|---|---|
| username | String | 用户名 |
| password | String | 明文密码 |

成功响应：

```json
{
  "success": true,
  "message": "登录成功",
  "userId": 1,
  "username": "admin01",
  "role": "admin",
  "studyPurpose": "四级",
  "experience": 120,
  "literacy": 3500
}
```

> **关键：** `role` 字段为 `"admin"` 时，前端首页应展示管理入口（例如侧边栏或顶栏增加"管理"菜单项）。

---

## 三、通用约定

### 3.1 管理员鉴权

系统使用 **HTTP Session** 进行身份认证。用户登录后，`LoginFilter` 自动验证会话状态。后端 Servlet 通过 `ServletUtil.getSessionUserId(request)` 获取当前登录用户 ID，并校验其 `role` 字段是否为 `admin`。

> **重要变更：** `adminUserId` 参数**不再需要**在请求中传递。前端只需确保浏览器自动携带 `JSESSIONID` Cookie（登录时由后端通过 `Set-Cookie` 响应头设置，`Path="/"`）。后端根据会话自动识别当前用户并校验管理员权限。

- **鉴权失败响应：**

```json
{
  "success": false,
  "message": "无管理员权限"
}
```

> **前端注意：** 收到此响应时应提示用户并跳转回首页或登录页。未登录时访问管理接口会收到 401 响应（由 `LoginFilter` 拦截）。

### 3.2 请求方式

| 接口 | 方法 | Content-Type |
|---|---|---|
| 用户列表 | GET | —（Query String） |
| 请求日志查询 | GET | —（Query String） |
| 在线用户查询 | GET | —（Query String） |
| 传统管理操作（创建/删除/重置密码/设置角色等） | POST | application/x-www-form-urlencoded |
| 部分新接口（更新文章等） | POST | application/json（JSON Body） |

> **注意：** 不同接口的 Content-Type 可能不同，请以各接口详述中的说明为准。新接口逐渐采用 JSON Body 格式。

### 3.3 响应格式

所有接口统一返回 JSON：

```json
{
  "success": true | false,
  "message": "操作结果描述"
}
```

部分接口会额外携带数据字段（如 `users` 数组、`logs` 数组、`onlineUsers` 数组等），详见各接口说明。

### 3.4 枚举值约束

**学习阶段 / 难度等级：** `初中` `高中` `四级` `六级` `考研` `托福`

**用户角色：** `normal` `vip` `admin`（仅 set-role 接口传 normal/vip，不能将用户设为 admin）

## 四、API 端点详述

### 4.1 用户管理

#### 4.1.1 获取全部用户列表

```
GET /api/admin/user/list
```

| 参数 | 必须 | 类型 | 说明 |
|---|---|---|---|
| — | — | — | 管理员身份由 Session 自动识别 |

成功响应：

```json
{
  "success": true,
  "users": [
    {
      "userId": 1,
      "username": "admin01",
      "role": "admin",
      "studyPurpose": "四级",
      "literacy": 5000,
      "experience": 200,
      "lastLogin": "2026-06-29 10:30:00",
      "createdAt": "2026-01-15 08:00:00"
    },
    {
      "userId": 2,
      "username": "zhangsan",
      "role": "normal",
      "studyPurpose": "六级",
      "literacy": 3200,
      "experience": 80,
      "lastLogin": "2026-06-28 18:00:00",
      "createdAt": "2026-03-20 14:00:00"
    }
  ]
}
```

**返回字段说明：**

| 字段 | 类型 | 说明 |
|---|---|---|
| userId | Long | 用户唯一 ID |
| username | String | 登录用户名 |
| role | String | 角色：`normal` / `vip` / `admin` |
| studyPurpose | String | 当前学习阶段 |
| literacy | Integer | 词汇量测试得分 (0-20000) |
| experience | Integer | 经验值 |
| lastLogin | String | 最近登录时间 (yyyy-MM-dd HH:mm:ss)，可能为 null |
| createdAt | String | 注册时间 |

> **前端建议：** 列表展示 userId、username、role、studyPurpose、experience、createdAt；lastLogin 和 literacy 可放到详情或 tooltip。role 可用不同颜色的 tag/badge 区分。

---

#### 4.1.2 创建用户

```
POST /api/admin/user/create
```

| 参数 | 必须 | 类型 | 说明 |
|---|---|---|---|
| username | 是 | String | 新用户名，≤16 字符 |
| studyPurpose | 是 | String | 学习阶段（见枚举约束） |

成功响应：

```json
{
  "success": true,
  "message": "用户创建成功"
}
```

可能失败响应：

| message | 原因 |
|---|---|
| 用户名不能为空 | username 缺失或空 |
| 用户名不能超过16个字符 | username 超长 |
| 用户名已存在 | 用户名重复 |
| 请选择学习阶段 | studyPurpose 缺失或空 |
| 无效的学习阶段 | studyPurpose 不在枚举范围内 |

> **重要：** 管理员创建的用户默认密码为 `123456`，默认角色为 `normal`。创建成功后前端应提示"初始密码为 123456"。

#### 4.3.3 强制用户下线

```
POST /api/admin/monitor/kick-user
```

Content-Type: `application/x-www-form-urlencoded`

| 参数 | 必须 | 类型 | 说明 |
|---|---|---|---|
| targetUserId | 是 | Long | 要强制下线的目标用户 ID |

成功响应：

```json
{
  "success": true,
  "message": "用户已被强制下线"
}
```

可能失败响应：

| message | 原因 |
|---|---|
| 缺少目标用户ID | targetUserId 缺失或无效 |
| 不能强制自己下线 | 管理员试图踢自己 |
| 用户不存在 | targetUserId 对应的用户不存在 |
| 不能强制管理员下线 | 目标用户也是管理员 |
| 该用户当前不在线 | 目标用户没有活跃会话 |

**强制下线机制：**

1. 目标 userId 加入 `kickedUserIds` 集合，会话从在线映射中移除
2. 目标用户下次发起任何 API 请求时，`LoginFilter` 检测到其已被标记
3. 目标用户收到 401 响应："您已被管理员强制下线，请重新登录"
4. 被踢用户前端检测到 401 → 清除本地状态 → 跳转登录页

> **前端建议：** 在在线用户列表中为每个非管理员用户添加"强制下线"按钮。点击前弹出二次确认对话框。操作成功后刷新在线用户列表。对 admin 角色用户隐藏此按钮。

## 五、前端页面设计建议

### 5.1 权限入口

登录后根据 `role` 字段判断：

```javascript
// 登录成功后存储用户信息
const user = loginResponse;
localStorage.setItem('userId', user.userId);
localStorage.setItem('role', user.role);

// 路由守卫或模板中
if (user.role === 'admin') {
  // 显示"管理中心"入口
}
```

### 5.2 管理页面路由建议

```
/admin               → 管理首页（仪表盘，可选）
/admin/users         → 用户管理页（列表 + 操作按钮）
/admin/articles      → 文章管理页（列表 + 新建 + 编辑 + 删除）
/admin/monitor/users → 在线用户监控页（实时列表 + 强制下线）
/admin/monitor/logs  → IP 请求日志页（后端过滤 + 分页）
```

### 5.3 用户管理页 (`/admin/users`)

**顶部操作栏：**
- "新建用户"按钮 → 弹出对话框：输入 username、选择 studyPurpose

**用户列表表格：**

| 列 | 说明 |
|---|---|
| ID | userId |
| 用户名 | username |
| 角色 | role（彩色 tag：admin=红色, vip=金色, normal=灰色） |
| 学习阶段 | studyPurpose |
| 经验值 | experience |
| 注册时间 | createdAt |

**每行操作按钮：**
- 设为 VIP / 设为普通 → 调用 set-role
- 重置密码 → 确认后调用 reset-password
- 删除 → 二次确认后调用 delete（管理员自己不可删）

### 5.4 文章管理页 (`/admin/articles`)

**顶部操作栏：**
- "新建文章"按钮 → 弹出对话框或跳转新页面

|---|---|---|

**文章列表：**
- 复用 `/api/article/list` 接口获取现有文章
- 每行增加"删除"按钮

### 5.5 交互规范建议

1. **危险操作确认：** 删除用户、删除文章、重置密码都应有二次确认弹窗
2. **操作反馈：** 成功/失败使用 toast 或 message 组件提示
3. **列表刷新：** CUD 操作成功后自动刷新列表
4. **错误处理：** 鉴权失败（"无管理员权限"）时，清除本地用户状态并跳转到登录页
5. **参数校验：** 前端应先做基本校验（非空、长度限制），减少无效请求

### 5.6 在线用户监控页 (`/admin/monitor/users`)

- 顶部显示在线总人数卡片（大号数字 + "人在线" 标签）
- 用户列表表格：userId、用户名、角色（彩色 tag）、学习阶段
- 每行操作按钮："强制下线"（admin 角色隐藏该按钮）
- 自动刷新开关（默认开启，30 秒间隔）
- 手动刷新按钮

### 5.7 IP 请求日志页 (`/admin/monitor/logs`)

- 顶部显示日志总数
- 筛选栏：IP 输入框（ipFilter 参数）、时间范围选择器（timeFrom / timeTo 参数）、用户名输入框（userFilter 参数）——均提交到后端进行过滤
- 日志表格：时间、IP、方法、路径、用户名
- 后端分页控件（page / pageSize 参数，显示 total / totalPages）
- "查询"按钮（触发筛选请求）、手动刷新按钮

## 七、注意事项

- 系统使用 **HTTP Session** 进行身份认证，`adminUserId` 参数**不再需要**在请求中传递。前端只需确保浏览器自动携带 `JSESSIONID` Cookie
- 传统管理操作（创建/删除/重置密码/设置角色/授予VIP/强制下线）使用 `application/x-www-form-urlencoded`；部分新接口（如更新文章）使用 `application/json`（JSON Body），请以各接口详述为准
- 管理员权限在每次请求时由 `LoginFilter` + `ServletUtil.getSessionUserId(request)` 实时校验
- 删除用户是**物理删除**，会级联清除投票记录和学习记录，不可恢复
- 删除文章是**物理删除**，会级联清除关联的题库数据，不可恢复
- 管理员不能通过 set-role 接口修改其他管理员的角色，也不能删除自己的账户
- 初始密码 `123456` 为明文，前端应在创建/重置成功后提示用户尽快修改密码
